## Neon Postgres in Stripe Projects

**Neon is now available** in [**Stripe Projects**](https://projects.dev). Stripe Projects is a **Stripe CLI** workflow for **hooking an app up to backends**: you pick services from a catalog (databases, hosting, auth, and more), provision them into **your** provider accounts, and get **connection strings and keys** into your environment so your code or your **agent** can run against a live database without dashboard copy-paste.

Example CLI flow (after the Stripe CLI is installed and you have run `stripe login`):

```bash
stripe projects init my-app   # set up your project
stripe projects catalog        # browse available services
stripe projects add neon/postgres       # provision a Neon database
```

Read more in [Neon works with Stripe Projects for agentic provisioning](/blog/neon-works-with-stripe-projects-for-agentic-provisioning).

## Neon Auth webhooks walkthrough with Resend

We [announced Neon Auth webhooks](/changelog/2026-03-13#neon-auth) a couple of weeks ago: HTTP notifications for auth events so you can own delivery (SMS, email, WhatsApp), validate signups, or sync to other systems.

This week we have a new guide, [**Customizing Neon Auth with Webhooks**](/guides/neon-auth-webhooks-nextjs), that shows a **Next.js** app that handles webhooks and sends OTP email with **Resend**, tests locally with **ngrok**, and uses a blocking handler to reject signups you don't want (for example disposable domains). Use it alongside our [Webhooks](/docs/auth/guides/webhooks) docs when you are ready to implement.

## AI agents and instant database branching

Neon's instant **database branching** and **AI coding agents** work well together. Branching gives each run an isolated Postgres copy in seconds, so agents can perfom scehma migrations, run tests, and try risky changes safely. The new guides below demonstrate this workflow with Codex and Claude Code.

- [**Safe AI-powered schema refactoring with OpenAI Codex and Neon**](/guides/openai-codex-neon-mcp): Use **OpenAI Codex CLI** with the **Neon MCP Server** so Codex can create an isolated branch, run Drizzle migrations, and validate schema refactors safely.

- [**Isolated Subagents: Running Claude Code in parallel with Neon Database Branching**](/guides/isolated-subagents-neon-branching): Use a `post-checkout` hook to give each **Claude Code subagent** its own Git worktree and **Neon branch**, so parallel agents can run without code or database collisions.

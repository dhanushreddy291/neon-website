---
title: Deprecations for Azure regions, pg_search, and account consumption API
---

## Neon Azure regions are deprecated

As of **April 2, 2026**, Neon no longer allows **new** projects in Azure. The regions `azure-eastus2`, `azure-westus3`, and `azure-gwc` are deprecated.

If you already run on Azure, we will email you about timelines for existing projects. Start planning your move now. **[Region migration](/docs/import/region-migration)** walks through options: another Neon region, a Postgres-compatible export, or **[Databricks Lakebase](/docs/guides/migrate-neon-to-lakebase)** if you need Postgres to stay on Azure (for example residency or colocation with other Azure workloads).

## pg_search on Neon is deprecated

Neon support for the **`pg_search`** extension is deprecated. As of **March 19, 2026**, it is not available for **new** Neon projects. If you already rely on it, start planning a migration. See **[The pg_search extension](/docs/extensions/pg_search)** for background, migration guidance, and alternatives.

<details>
<summary>**Neon API**</summary>

- The **`GET /consumption_history/account`** endpoint ([Get account consumption metrics](https://api-docs.neon.tech/reference/getconsumptionhistoryperaccount)) is **deprecated**, with a planned sunset of **June 1, 2026**. If you use aggregated account-level consumption metrics, move to **[project-level consumption metrics](/docs/guides/consumption-metrics)** or another supported endpoint for your plan before that date.

</details>
